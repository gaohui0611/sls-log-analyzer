**用例1：业务逻辑分析**

```

请分析是否存在异常的用户行为，如频繁请求、非法参数、恶意攻击等。

查询关键词：ERROR

```

## 💡 使用建议

### 自定义Prompt最佳实践

1. **明确分析目标**
   ```
   ✅ 好：请分析用户登录失败的原因，重点关注验证逻辑
   ❌ 差：分析一下
   ```

2. **指定关注点**
   ```
   ✅ 好：请重点分析数据库查询性能和缓存命中率
   ❌ 差：看看有什么问题
   ```

3. **提供上下文**
   ```
   ✅ 好：这是支付接口的日志，请分析是否存在支付失败但未回滚的情况
   ❌ 差：分析支付问题
   ```

### 日志详情使用技巧

1. **结合TraceID追踪**
   - 复制TraceID到查询关键词
   - 查看完整的请求链路

2. **关注时间分布**
   - 查看日志时间戳
   - 分析问题发生的时间规律

3. **对比不同用户**
   - 查看UserID
   - 分析是否是特定用户的问题
