# Bug修复报告：前端导航点击事件绑定问题

## 问题描述

前端 `app.js` 中存在多个函数使用内联 `onclick` 属性绑定，但这些函数没有暴露到全局作用域（window对象），导致点击按钮时出现 `function is not defined` 错误。

## 受影响的功能

1. **项目管理**
   - 编辑项目按钮 (`editProject`)
   - 删除项目按钮 (`deleteProject`)

2. **报告管理**
   - 查看报告详情按钮 (`viewReport`)
   - 删除报告按钮 (`deleteReport`)
   - 分页导航按钮 (`goToReportPage`)

3. **AI配置模态框**
   - 打开编辑模态框 (`openEditAiModal`)
   - 关闭模态框 (`closeEditAiModal`)
   - 保存配置 (`saveEditAiConfig`)
   - 测试配置 (`testEditAiConfig`)

## 根本原因

在 `renderProjects()` 和 `renderReports()` 等函数中，动态生成的HTML使用了内联 `onclick` 属性：

```javascript
// 错误示例
<button onclick="editProject('${id}')">编辑</button>
```

但对应的函数定义为局部函数：

```javascript
// 局部函数，无法从HTML访问
async function editProject(id) { ... }
```

## 解决方案

将所有需要从HTML内联事件访问的函数暴露到全局作用域：

```javascript
// 修复后：暴露到window对象
window.editProject = async function(id) { ... }
window.deleteProject = async function(id) { ... }
window.viewReport = async function(id) { ... }
window.deleteReport = async function(id) { ... }
window.goToReportPage = function(page) { ... }
window.openEditAiModal = function() { ... }
window.closeEditAiModal = function() { ... }
window.saveEditAiConfig = async function() { ... }
window.testEditAiConfig = async function() { ... }
```

2. ✅ 报告列表中的"查看详情"和"删除"按钮可以点击
3. ✅ 报告分页导航按钮可以点击

## 建议改进

为了避免类似问题，建议：

1. **使用事件委托**：在父元素上绑定事件监听器，而不是使用内联onclick
2. **避免内联事件**：使用 `addEventListener` 代替 `onclick` 属性
3. **代码审查**：检查所有动态生成的HTML，确保事件绑定正确

### 示例改进方案

```javascript
// 推荐方式：使用事件委托
document.getElementById('projectList').addEventListener('click', (e) => {
    if (e.target.matches('.edit-btn')) {
        const id = e.target.dataset.id;
        editProject(id);
    }
    if (e.target.matches('.delete-btn')) {
        const id = e.target.dataset.id;
        deleteProject(id);
    }
});

// HTML中使用data属性
<button class="edit-btn" data-id="${id}">编辑</button>
```
